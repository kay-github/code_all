## 1. 产品背景与定位

### 1.1 用户问题

常规行情软件能够展示一只股票的涨跌幅，但普通用户仍难以快速回答：

- 这只股票今年以来到底表现如何？
- 它跑赢了多少只 A 股？
- 它大约处于市场前百分之多少？
- 同期沪深300表现如何？

### 1.2 一句话定位

输入任意 A 股名称或代码，快速查看其年初至今收益，并判断它跑赢了多少只可比股票、处于市场什么位置。

### 1.3 核心价值

产品的核心不是重复展示一个 YTD 数字，而是把收益率翻译成易懂的市场位置：

> 某股票年初至今上涨 XX.XX%，跑赢 XXXX 只其他可比股票，占可比样本的 XX.XX%，约位于市场前 XX.XX%。

### 1.4 设计原则

- 可信：所有结果必须能够追溯到明确的日期、股票池和计算口径。
- 克制：一个搜索框、一张主结果卡、一张基准卡，避免交易终端式堆叠。
- 一致：目标股票、全市场股票和指数必须基于同一完整市场统计日。
- 易懂：优先使用自然语言、数量和百分比，不用复杂图表代替结论。
- 移动优先：首要体验以手机单手操作和快速读取为准。

## 2. 目标与非目标

### 2.1 MVP 目标

- 支持按股票部分名称、完整名称、部分代码或完整代码检索。
- 用户从下拉候选中选择股票后自动计算，无需再次提交。
- 展示个股 YTD、跑赢数量、跑赢比例和市场前百分比。
- 默认与沪深 A 股比较，可即时选择加入北交所 A 股。
- 结果区最后展示沪深300价格指数 YTD。
- 明确展示数据截止日、其他可比股票总数 peerCount 和统计口径。
- 完整覆盖加载、无结果、新股、停牌、过期数据和服务失败状态。

### 2.2 MVP 非目标

- 不做盘中实时排名。
- 不做买卖建议、目标价、荐股或投资评分。
- 不做登录、自选股、提醒和账户体系。
- 不做 K 线、复杂走势图或收益分布图。
- 不做行业、市值、ST 排除等高级筛选。
- 不做月度、季度、近一年或历史年份切换。
- 不做“跑赢沪深300多少个百分点”，避免不同收益口径被错误直接比较。

### 2.3 后续迭代候选

- 相对沪深300全收益指数的同口径超额收益。
- 行业内排名、板块内排名、市值分组排名。
- 近一月、近三月、近一年等周期。
- 排除 ST、长期停牌等高级筛选。
- 最近查询、自选列表和可分享结果卡。
- 历史日期、历史年份及排名变化。
- 明确标识的盘中估算模式。

## 3. 目标用户与用户故事

### 3.1 目标用户

- 希望快速判断个股年度表现的普通投资者。
- 需要进行简单横向比较的研究人员或内容创作者。
- 已知股票名称或代码，但不希望在复杂行情软件中层层查找的用户。

### 3.2 核心用户故事

1. 作为用户，我可以输入“新易盛”“易盛”“300502”或“3005”，找到并选择明确股票。
2. 作为用户，我选择股票后，可以立即看到其年初至今收益。
3. 作为用户，我可以看到该股票跑赢了多少只其他可比股票，以及对应比例。
4. 作为用户，我可以选择是否将北交所 A 股加入比较池。
5. 作为用户，我可以知道结果使用了哪个交易日、多少只样本以及什么口径。
6. 作为用户，我可以在结果最后看到同期沪深300 YTD。
7. 当股票为当年新股、处于停牌或数据服务异常时，我能够得到明确且不误导的说明。

## 4. 统计与数据口径

本节属于强约束。任何实现不得用更方便的数据替代而不修改本文档。

### 4.1 时间口径

| 名称 | 定义 |
| --- | --- |
| 基准日 b | 上一自然年最后一个完整交易日 |
| 统计日 t | 沪深市场、可选北交所及沪深300数据均已完整的最近共同交易日 |
| 页面日期 | 默认显示“截至 YYYY-MM-DD 收盘” |
| 盘中行为 | 继续展示最近完整交易日数据，不混入盘中实时价格 |
| 周末及节假日 | 展示最近完整交易日快照 |

禁止使用目标股票盘中实时价格与全市场上一交易日收盘价混排。

### 4.2 个股 YTD

主口径采用复权收益，处理分红、送转、拆并股等公司行为：

    adjustedPrice = close × adjFactor
    YTD = adjustedPrice(t) ÷ adjustedPrice(b) - 1

要求：

- 内部使用完整精度计算。
- 基准端取不晚于基准日 b 的最近有效复权收盘价，统计端取不晚于统计日 t 的最近有效复权收盘价；如任一端不存在有效价格，则不进入可比池。
- 页面按百分比显示两位小数，并始终带正负号。
- 上涨显示“+XX.XX% 上涨”，下跌显示“-XX.XX% 下跌”，零显示“0.00% 持平”。
- 不因页面显示四舍五入而改变排名。

### 4.3 默认可比股票池

默认股票池为当前上市的沪深 A 股：

- 包含上交所、深交所主板。
- 包含创业板和科创板。
- 默认不含北交所；用户开启后加入符合条件的北交所 A 股。
- ST 与 *ST 股票默认包含。
- 排除 B 股、基金、ETF、REIT、可转债、存托凭证、指数和已退市证券。
- 必须基于证券主数据中的交易所和证券类型判断，不得仅凭代码前缀猜测。

### 4.4 可比样本资格

一只股票进入 YTD 比较池必须同时满足：

1. 当前仍处于上市状态。
2. 在基准日或之前已经上市。
3. 基准端存在有效收盘价和复权因子。
4. 统计端存在有效收盘价和复权因子。
5. 数据未被质量校验判定为异常。

页面所称“市场”实际指“当前上市且具有完整 YTD 的可比 A 股”，统计说明中必须明确。

### 4.5 当年新股

- 在基准日之后上市的股票没有严格意义上的 YTD。
- 当年新股不进入 YTD 比较池。
- 用户主动查询时显示“年初尚未上市，暂无完整 YTD”。
- 如数据允许，可以展示“上市以来收益”，但不得标成 YTD，也不得展示市场 YTD 排名。

### 4.6 停牌股票

- 停牌股票默认保留在可比池中。
- 统计端采用不晚于统计日的最近有效收盘价。
- 如果目标股票最近有效价格日期早于市场统计日，必须同时显示：
  - 市场排名截至日期。
  - 该股票最近有效收盘价日期。
- 如果目标股票基准端实际价格日期早于基准日，统计说明中必须同时显示基准日和 basePriceDate，不得将更早价格伪装成基准日当日价格。
- 不得在前端静默剔除停牌股票。

### 4.7 跑赢数量与比例

若目标股票属于当前比较池，先从比较池中排除目标股票自身。

    跑赢数量 W = YTD 严格低于目标股票 YTD 的其他可比股票数量
    其他可比股票总数 N = 排除目标股票自身后的有效样本数
    跑赢比例 P = W ÷ N × 100%
    更高收益股票数 H = YTD 严格高于目标股票 YTD 的其他可比股票数量
    严格名次 R = H + 1
    排名总体 T = N + 1
    市场前百分比 Q = R ÷ T × 100%

规则：

- 收益率相等的股票不计为被跑赢。
- 收益率相等的股票共享同一严格名次。
- 排名使用未四舍五入的 YTD。
- 数量显示千位分隔符。
- 比例显示两位小数。
- “市场前百分比”前使用“约”，降低并列收益带来的精确名次误解。
- 页面“其他可比股票总数”或“有效对比样本”始终显示 N，并作为跑赢比例的唯一分母。
- 股票池原始有效数量称为 poolEligibleCount；目标属于比较池时，poolEligibleCount = N + 1，否则 poolEligibleCount = N。poolEligibleCount 不作为页面跑赢比例分母。
- 若目标为北交所股票且北交所开关关闭，可以将该目标与沪深 A 股比较；页面必须明确比较范围为沪深 A 股。

推荐结果文案：

> 跑赢 XXXX / YYYY 只其他可比股票，占沪深 A 股可比样本的 AA.AA%，约位于市场前 BB.BB%。

以上数字仅作为文案示意，不代表真实行情。

### 4.8 北交所开关

- 首次访问默认关闭。
- 关闭文案为“比较范围：沪深 A 股”。
- 开启文案为“比较范围：沪深 A 股及北交所 A 股”。
- 开关仅改变跑赢数量、跑赢比例、市场前百分比和其他可比股票总数 peerCount。
- 开关不得改变个股 YTD、统计日期和沪深300 YTD。
- 设置仅在当前访问期间保留；新访问恢复默认关闭。
- 北交所关闭时不得使用“跑赢全市场”文案。

### 4.9 沪深300

- MVP 展示大众熟悉的沪深300价格指数，标识为“沪深300（价格指数）”。
- 指数 YTD = 统计日指数收盘点位 ÷ 基准日指数收盘点位 - 1。
- 指数采用与市场快照一致的基准日和统计日。
- 沪深300位于结果区最后，视觉权重低于个股结果。
- 由于个股采用复权收益、沪深300为价格指数，MVP 不计算两者之间的超额收益。
- 如后续增加严格同口径的超额收益，应改用沪深300全收益指数并更新本文档。

### 4.10 数据质量与快照

- 每个交易日收盘后批量生成一次完整市场快照，建议在 18:00 后执行。
- 正常发布时间阈值固定为交易日 18:30（Asia/Shanghai）；如实际调度策略调整，必须同步更新本文档。
- expectedAsOf 由服务端交易日历计算：交易日 18:30 前为上一个交易日，18:30 起为当日；周末和节假日为最近一个交易日。
- 当 asOf 早于 expectedAsOf 时，isStale 为 true；周末、节假日或交易日盘中展示上一完整交易日不视为过期。
- 快照包含 asOf、expectedAsOf、publishedAt、isStale、baseDate、口径版本、股票池版本、poolEligibleCount 与 excludedCount。
- 沪深和“沪深加北交所”分别生成可查询的预排序数据。
- 批次通过完整性校验后原子发布，禁止用户读取半更新数据。
- 更新失败时保留上一份完整快照，并明确展示实际数据日期。
- 每日检查股票池数量突变、复权因子缺失、共同日期不一致和疑似异常收益。
- 生产数据优先使用一个能够统一覆盖证券主数据、日线、复权因子、交易日历和指数的合规数据源。
- AkShare 或非正式网页接口可以用于原型验证，不宜作为唯一生产数据源。

## 5. 信息架构与页面流程

### 5.1 页面顺序

1. 工具站顶部导航或品牌区。
2. 页面标题和一句说明。
3. 股票搜索框。
4. 比较范围设置。
5. 初始提示或个股结果主卡。
6. 沪深300同期基准卡。
7. 统计口径与数据说明。
8. 免责声明。

### 5.2 标准用户流程

1. 用户进入页面，北交所默认关闭。
2. 用户输入部分名称或代码。
3. 系统延迟约 200 至 300 毫秒搜索并展示候选。
4. 用户从候选中选择明确股票。
5. 下拉关闭，移动端软键盘收起，结果区显示固定高度骨架。
6. 系统返回个股 YTD、跑赢数量、比例、peerCount 和沪深300 YTD。
7. 用户切换“纳入北交所”。
8. 页面仅更新市场排名区域，个股 YTD和沪深300保持不动。
9. 用户可以继续搜索并选择另一只股票。

## 6. 功能需求

### FR-01 股票搜索（必须）

- 搜索范围覆盖上交所、深交所和北交所当前上市 A 股；“纳入北交所”开关只影响比较池，不影响搜索范围。
- 支持完整代码、部分代码、完整中文名称和部分中文名称。
- 中文输入 1 个字符或数字输入 2 位后可以发起搜索。
- 搜索防抖建议 200 至 300 毫秒。
- 最多直接展示 8 条候选，超出后允许滚动。
- 排序优先级：精确代码、完整名称、名称前缀、其他包含匹配。
- 每条候选必须展示名称、代码、交易所和板块。
- 匹配字符可以加粗或使用品牌色，但不得只依赖颜色。
- 用户必须选择明确候选，不得根据模糊输入自动取第一条。
- 支持清空、重新输入、鼠标或触控选择。
- 支持方向键上下选择、Enter 确认、Esc 关闭和 Tab 离开。

推荐占位文案：

> 输入股票名称或代码，如：新易盛 / 300502

推荐辅助文案：

> 请从下拉结果中选择股票。

### FR-02 自动计算（必须）

- 选择股票后自动计算，不设置额外“查询”按钮。
- 首次查询显示结果卡骨架，不使用整页加载。
- 更换股票时保留旧卡并显示“正在更新”，避免闪白。
- 只接受最后一次选择或设置对应的结果，忽略更早请求的延迟返回。
- 完成后原位替换，不弹成功 Toast，不强制滚动或移动键盘焦点。
- 自动计算状态通过礼貌级实时区域向读屏用户播报。

### FR-03 比较范围（必须）

- 采用即时生效的“纳入北交所”开关。
- 开关整行均可点击，触控区域不小于 44px 高。
- 未选择股票时只更新页面状态，不发起无意义请求。
- 已有结果时只更新排名区。
- 快速连续切换时以最后状态为准。
- 更新失败时保留原比较结果，并在排名区提供重试。

### FR-04 个股结果（必须）

必须展示：

- 股票名称。
- 股票代码、交易所或板块。
- 数据统计日期。
- 年初至今收益及上涨、下跌或持平文字。
- 跑赢比例。
- 跑赢数量和其他可比股票总数。
- 约位于市场前百分比。
- 当前比较范围。
- 其他可比股票总数 peerCount。

### FR-05 沪深300基准（必须）

- 在个股主结果之后展示。
- 标题为“同期基准”。
- 名称为“沪深300（价格指数）”。
- 展示同期 YTD 和数据日期。
- 独立加载、独立失败；指数数据失败不得导致个股结果整体失败。
- 数值字号和卡片尺寸必须低于个股主结果。

### FR-06 统计口径说明（必须）

- 页面提供“统计口径与数据说明”入口。
- 移动端可以使用可展开区域或底部抽屉。
- 不得只通过桌面悬停 Tooltip 展示。

## 13. 数据与接口边界

### 13.1 架构原则

- 前端始终请求同源后端接口。
- 数据供应商密钥只保存在服务端环境变量中。
- 不允许前端直接连接需要密钥的数据服务。
- 不在每次用户查询时临时抓取并计算五千多只股票。
- 日终任务负责生成完整市场快照，线上查询只读取快照和缓存。
- 搜索主数据每日更新并缓存。

### 13.2 建议接口

接口名称可在实现评审中调整，但返回语义不得改变。

股票搜索：

    GET /api/stock-search?q=新易

建议返回：

    items: [
      {
        symbol,
        code,
        name,
        exchange,
        board,
        listingStatus
      }
    ]

YTD 与市场排名：

    GET /api/stock-ytd?symbol=300502.SZ&includeBse=false

建议返回：

    {
      asOf,
      expectedAsOf,
      publishedAt,
      isStale,
      baseDate,
      methodologyVersion,
      stock: {
        symbol,
        name,
        exchange,
        board,
        ytd,
        direction,
        basePriceDate,
        lastPriceDate,
        isSuspended,
        hasFullYtd,
        ineligibilityReason,
        sinceListingReturn
      },
      comparison: {
        scope,
        includeBse,
        beatCount,
        peerCount,
        beatRatio,
        higherCount,
        rankPosition,
        rankPopulation,
        topRatio,
        poolEligibleCount,
        excludedCount
      },
      benchmark: {
        symbol,
        name,
        type,
        ytd,
        asOf
      }
    }

ineligibilityReason 采用稳定枚举，前端根据枚举映射用户文案：

| 枚举值 | 含义 | 页面处理 |
| --- | --- | --- |
| NEW_LISTING | 基准日后上市 | 显示“暂无完整 YTD”，可展示上市以来收益 |
| MISSING_BASE_PRICE | 缺少基准端有效价格 | 显示“缺少年初可比价格，暂无可比 YTD” |
| MISSING_CURRENT_PRICE | 缺少统计端有效价格 | 显示“缺少当前可比价格，暂无可比 YTD” |
| MISSING_ADJ_FACTOR | 复权因子不完整 | 显示“复权数据不完整，暂无可比 YTD” |
| DATA_QUALITY_REJECTED | 数据质量校验未通过 | 显示“数据暂未通过校验，暂无可比 YTD” |
| NOT_ELIGIBLE_SECURITY | 当前证券不属于支持的 A 股范围 | 显示“不属于当前支持的证券范围” |

以上属于数据资格结果，不等同于接口调用失败；前端不得提供会反复得到同一结果的无意义“重试计算”。

### 13.3 性能目标

- 搜索输入防抖：200 至 300ms。
- 缓存命中时，搜索接口 P95 目标不高于 500ms。
- 缓存命中时，结果接口 P95 目标不高于 1 秒。
- 页面切换北交所范围时优先复用快照，不重新抓取供应商数据。
- 首屏和结果区不得因等待接口而发生明显布局偏移。

### 13.4 数据源选择要求

数据源职责锁定如下：

| 数据源 | 职责 | 是否可单独发布正式快照 |
| --- | --- | --- |
| 东方财富 | 全市场最新 f25 YTD 参考锚点、用户认知对齐、在线哨兵 | 否；两位小数精度不足以作为高精度排名键 |
| Tushare Pro | 证券主数据、交易日历、原始收盘价、复权因子、沪深300；全市场独立计算与整源备用 | 是；前提是完整性和质量闸门通过 |
| 腾讯财经 | 单股前复权 K 线、分红送股样本和异常股票复核 | 否；不承担五千只股票的全市场批量源 |
| 上一份已发布快照 | 东财和 Tushare 同时不可用时维持服务连续性 | 是；必须显示真实 asOf 和 isStale |

约束：

- publishedYtd 优先使用 close × adjFactor 两端比值计算的 computedYtd。
- 东财 f25 标准化为 referenceYtd，并与 computedYtd 分开保存，不互相覆盖。
- 多源结果不得求平均，也不得把某源原始价格与另一源复权因子混算。
- computedYtd 与 referenceYtd 差值不超过 5bp 时通过；大于 5bp 至 20bp 时告警；大于 20bp 时隔离该股票。
- reference-only 只允许原型或诊断，默认不得发布生产快照。
- 东财整体失败时只能切换到一份完整的 Tushare 快照，不做大规模逐股拼源。
- 腾讯仅用于异常仲裁；腾讯单独可用时不能生成新的全市场正式快照。
- 股票池必须以独立证券主数据定义，再与东财行情相交；不得把东财列表中的定向转让等非普通 A 股直接纳入排名。
- AkShare 中转的东财接口不算独立数据源。
- 生产使用前必须确认供应商商用授权、调用额度和服务条款。

详细接口、实测证据、容灾流程和当前限制见同目录 DATA_SOURCES.md。

## 14. MVP 验收标准

### 14.1 功能验收

- [ ] 输入完整股票代码可以找到准确股票。
- [ ] 输入部分代码可以展示相关候选。
- [ ] 输入完整或部分中文名称可以展示相关候选。
- [ ] 每条候选都显示名称、代码、交易所和板块。
- [ ] 未明确选择候选时不会自动计算。
- [ ] 选择股票后自动计算，无额外提交按钮。
- [ ] 默认比较范围为沪深 A 股，不含北交所。
- [ ] 开启北交所后，个股 YTD和沪深300不变。
- [ ] 开启北交所后，跑赢数量、比例和 peerCount 按新范围更新。
- [ ] 结果展示个股 YTD、跑赢数量、跑赢比例和市场前百分比。
- [ ] 结果展示其他可比股票总数 peerCount、比较范围和数据日期。
- [ ] 沪深300位于结果区最后。
- [ ] 沪深300标识为价格指数。
- [ ] 沪深300失败不会导致个股结果失败。

### 14.2 口径验收

- [ ] 所有股票与指数使用同一统计截止日；停牌股票按最近有效价格处理，并记录 basePriceDate 和 lastPriceDate。
- [ ] 不混入目标股票盘中实时价格。
- [ ] 目标股票自身从比较分母中排除。
- [ ] 收益率相同的股票不计为被跑赢。
- [ ] 收益率相同的股票共享同一严格名次，市场前百分比按 rankPosition ÷ rankPopulation 计算。
- [ ] 排名使用未四舍五入值。
- [ ] 跑赢比例等于跑赢数量除以其他可比股票总数。
- [ ] peerCount 是页面显示的有效对比样本和跑赢比例分母；poolEligibleCount 不被误用为分母。
- [ ] 北交所关闭时不出现“全市场”文案。
- [ ] 当年新股不会显示为标准 YTD 或进入 YTD 排名。
- [ ] 停牌股票显示最近有效价格日期。
- [ ] 数据不完整的目标股票返回 ineligibilityReason，不展示 YTD 排名。
- [ ] 服务端按照交易日历和 18:30 阈值返回 expectedAsOf、isStale 和 publishedAt。
- [ ] 统计说明明确复权口径和有效股票池。

### 14.3 UI 验收

- [ ] YTD 是页面唯一超大数字。
- [ ] 金融数据区域中红色只表达上涨、绿色只表达下跌，蓝色表达交互和排名；错误红只用于错误状态。
- [ ] 涨跌同时有颜色、正负号和文字。
- [ ] 主卡与沪深300卡具有明显视觉层级差异。
- [ ] 首次加载使用固定高度骨架，页面不明显跳动。
- [ ] 切换北交所只更新排名区。
- [ ] 自动计算成功不弹冗余成功 Toast。
- [ ] 错误信息常驻于错误区域并提供重试。
- [ ] 320px 宽屏没有横向滚动。
- [ ] 浏览器放大至 200% 仍能完成核心流程。
- [ ] 所有触控区域不小于 44×44px。
- [ ] 键盘可以完成股票搜索和选择。
- [ ] 焦点状态清晰可见。

### 14.4 数据与安全验收

- [ ] 前端不包含数据供应商密钥。
- [ ] 用户查询读取预生成快照，不逐次全市场抓取。
- [ ] 东财 f25 作为 referenceYtd 保存，不能直接覆盖独立计算值。
- [ ] 分红、送股测试使用复权价格或价格与复权因子两端比值，未复权收益不能进入排名。
- [ ] baseAdjFactorDate、lastAdjFactorDate 分别与对应价格日期一致。
- [ ] computedYtd 与 referenceYtd 的 5bp 告警和 20bp 隔离门槛生效。
- [ ] reference-only 快照默认不可生产发布。
- [ ] 数据源切换按整份快照执行，不对全市场结果做多源平均。
- [ ] 股票池经过证券主数据过滤，不包含定向转让、基金、B 股等非目标证券。
- [ ] expectedUniverseCount 来自独立证券主数据；缺失时快照不得生产发布。
- [ ] expectedBaseDate 和 expectedAsOf 来自交易日历；缺失时快照不得生产发布。
- [ ] 99.8% 覆盖率、主数据校验和复权审计不能通过配置降级后继续生产发布。
- [ ] 被质量闸门阻断的 Candidate 快照不能进入用户查询。
- [ ] 快照带有统计日、基准日、口径版本和股票池版本。
- [ ] 快照校验通过后才原子发布。
- [ ] 更新失败时仍能显示上一份完整快照和真实日期。
- [ ] 数据异常不会导致半更新结果对外展示。

## 15. 必测场景

1. 正常沪深股票，YTD 为正。
2. 正常沪深股票，YTD 为负。
3. YTD 恰好为零。
4. 目标股票收益为市场最高或最低附近。
5. 多只股票收益完全相同。
6. 当年上市新股。
7. 当前停牌股票。
8. 基准日附近停牌但存在更早有效价格的股票。
9. 北交所股票作为目标，北交所开关关闭。
10. 北交所股票作为目标，北交所开关开启。
11. 搜索相似名称和同字股票。
12. 连续快速选择不同股票，较早请求后返回。
13. 快速连续切换北交所开关。
14. 搜索无结果。
15. 股票搜索接口失败。
16. 个股结果接口失败。
17. 北交所范围更新失败。
18. 沪深300数据单独失败。
19. 市场快照未按时更新。
20. 周末、节假日和交易日盘中访问。

典型业务验证股票可以使用“新易盛（300502.SZ）”，但验收时不得把文档示意数字当作固定预期值。

## 16. 防走偏清单

以下规则未经产品评审不得改变：

1. 第一版只使用完整收盘快照，不做盘中实时排名。
2. 默认股票池是沪深 A 股，北交所默认关闭。
3. 目标股票自身不计入比较分母。
4. 当年新股不包装为标准 YTD。
5. YTD 是页面唯一超大数字。
6. 排名使用蓝色表达；金融数据区域红涨绿跌，错误红只用于错误状态。
7. 沪深300必须位于个股结果之后。
8. MVP 不计算个股相对沪深300的超额收益。
9. 用户切换北交所时只更新排名区。
10. 数据日期、peerCount 和口径不能隐藏。
11. 前端不得直连需要密钥的数据服务。
12. 不为了视觉丰富增加无助于决策的环形图、K线或动画数字。
